# Bruno Converters

Use converter tooling when the user already has a machine-readable source and a broad first draft is more valuable than hand-authoring every request.

## Supported Official Inputs

- Postman collections
- Postman environments
- Insomnia collections
- OpenAPI specifications
- WSDL files

Bruno's official package is `@usebruno/converters` and the documented entry points are:

- `postmanToBruno(postmanCollection)`
- `postmanToBrunoEnvironment(postmanEnvironment)`
- `insomniaToBruno(insomniaCollection)`
- `openApiToBruno(openApiSpec)`
- `wsdlToBruno(wsdlContent)`

## When To Prefer A Converter

- The source material is authoritative and already structured.
- The API is large enough that manual transcription would be slow or error-prone.
- The user wants quick coverage first and is comfortable with cleanup afterward.

## When To Author By Hand

- The user only needs a few curated requests.
- The source material is incomplete, inconsistent, or mostly prose.
- The collection should emphasize business workflows rather than exhaustively mirroring the source spec.

## After Conversion

- Review filenames and folder grouping.
- Replace literal hosts with `{{baseUrl}}` or other environment variables.
- Move secrets into environment files or placeholder variables.
- Remove noisy requests, duplicated headers, and weak autogenerated tests.
- Align the converted files with the collection's existing target format before finalizing the edit.
